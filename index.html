Prework Study Guide

HTML

  • HTML is an acronym for "Hypertext Markup Language".
  • It is used as the structure of a webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • P inside of <> will create a paragraph for user to see.
  • Some styling elements within <> also include "H1,2,3,4,etc." for types of headings, "ul" for unordered lists like the one you're reading right now
    "br" is a line break like you just read, "ol" is an ordered list.
  • "li" is used for new lines in a list.
  • While adding images use <> "img" src="image file/link" and it helps to add an alt="text" to have a description of the image.
  • Adding links uses an anchor, like "< A href="url of link" >title of link(that user sees)< /A >.

CSS

  • CSS is an acronym for "Cascading Stylesheet".
  • There are 3 ways to style a webpage using CSS.
    1. Inline Styling - used on the html file itself, and only applies to the
      line or <> it is within, like I used to indent this ordered list.
    2. Internal Styling - used on the html file itself as well, but it is used by
      adding a < style > element in the < head > portion and is applied to the whole file.
    3. External Styling - this is an external CSS file with it's style rules that is linked to
      the html file with < link rel="stylesheet " href=" location" >.
  • An example of an external style rule is using a item, with
    the property you want to apply the value of the property. This looks kind of like
    this- item { color: red;}.
  • the propertied within {} are know as the "declaration".
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • Git status: checks what branch we are currently on.
  • Git checkout -b branch-name: creates a new branch and switches to it.
    this can also be used without -b and with an existing branch, which will let you switch to it.
  • Using "feature/new branch name" while creating a branch helps indicate a new feature.
  • Using "git add -A will add all modifications in working directory to staging area to be commited
    the -A indicates add all changes.
  • Always remember to sync with others before a pull request.
    this can be done by using "git pull origin "name of main branch"
  • Git commit -m "" allows for a message about the commit being made while adding a commit.
  • "Git pull origin main" will recieve branches made to local origin - source of pull will be
    in github repository.
  • Git push origin "branch name" will push local changes to remote branch.
