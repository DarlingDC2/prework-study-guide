HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • P inside of <> will create a paragraph for user to see
  • Some styling elements within <> also include "H1,2,3,4,etc." for types of headings, "ul" for unordered lists like the one you're reading right now
    "br" is a line break like you just read, "ol" is an ordered list.
  • "li" is used for new lines in a list
  • While adding images use <> "img" src="image file/link" and it helps to add an alt="text" to have a description of the image
  • Adding links uses an anchor, like "< A href="url of link" >title of link(that user sees)< /A >

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
    this can also be used without -b and with an existing branch, which will let you switch to it.
  • using "feature/new branch name" while creating a branch helps indicate a new feature.
  • using "git add -A will add all modifications in working directory to staging area to be commited
    the -A indicates add all changes.
  • always remember to sync with others before a pull request
    this can be done by using "git pull origin "name of main branch"
  • git commit -m "" allows for a message about the commit being made while adding a commit
  • "git pull origin main" will recieve branches made to local origin - source of pull will be
    in github repository
  • git push origin "branch name" will push local changes to remote branch
